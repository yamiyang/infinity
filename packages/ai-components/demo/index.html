自然语言驱动的
Web Components

用一行 <ai-component p="..."> 标签，让 LLM 流式生成并实时渲染任意 HTML 片段。纯 Web Component，可在 React / Vue / Svelte / 原生 HTML 中使用。

快速开始 → 🎮 在线体验

📦 安装

npm / pnpm / yarn
# 选择你的包管理器
npm install ai-components
pnpm add ai-components
yarn add ai-components
CDN（<script> 标签）
<script src="https://unpkg.com/ai-components/dist/ai-components.iife.global.js"></script>
💡 提示：IIFE 产物会自动注册 AIC 全局变量并调用 defineAIComponent()，你只需 AIC.configure() 即可使用。

⚡ 快速开始

推荐 方式一：ESM 模块 

import { configure, defineAIComponent } from 'ai-components';

// 1. 提供 LLM 请求函数（必须返回 AsyncIterable<string>）
configure({
  request: async function*(prompt, options) {
    // 调用你的 LLM API，逐 chunk yield 文本
    const res = await fetch('/api/chat', {
      method: 'POST',
      body: JSON.stringify({ prompt }),
      signal: options?.signal,
    });
    const reader = res.body.getReader();
    // ...解析 SSE 并 yield content
  }
});

// 2. 注册 <ai-component> 自定义元素
defineAIComponent();

方式二：<script> 标签

<script src="ai-components/dist/ai-components.iife.global.js"></script>
<script>
  AIC.configure({ request: yourRequestFn });
</script>

然后在 HTML 中使用

<!-- 只需设置 p 属性，组件会自动调用 LLM 并流式渲染 -->
<ai-component p="生成一个精美的用户注册表单"></ai-component>

<!-- 更新 p 属性会自动重新生成 -->
<ai-component p="一个天气预报卡片"></ai-component>

🔧 配置 LLM

configure() 是使用组件前的必要步骤。你需要提供一个 LLMRequestFn，它接收 prompt 和 options，返回 AsyncIterable<string>。 

type LLMRequestFn = (
  prompt: string,
  options?: { signal?: AbortSignal; systemPrompt?: string }
) => AsyncIterable<string>;

// 示例：接入 OpenAI 兼容 API
async function* openaiRequest(prompt, options) {
  const res = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o',
      messages: [
        { role: 'system', content: options?.systemPrompt || '' },
        { role: 'user', content: prompt },
      ],
      stream: true,
    }),
    signal: options?.signal,
  });

  // 解析 SSE 流...
  const reader = res.body.getReader();
  // ... yield 每个 delta.content
}

configure({ request: openaiRequest });

🏷️ <ai-component> 组件

属性 类型 说明
p string 必填。自然语言提示词。设置后组件自动调用 LLM 并流式渲染结果。修改后重新生成。

生命周期

connected 组件挂载时读取 p 属性，自动开始生成。先显示加载态（spinner + prompt 预览），流开始后切换到内容模式。
attribute p 属性变化时，取消当前生成并重新执行。
disconnected 从 DOM 移除时自动取消正在进行的生成（通过 AbortController）。

动态创建

const el = document.createElement('ai-component');
el.setAttribute('p', '生成一个天气卡片');
document.body.appendChild(el);

// 更新 prompt → 自动重新生成
el.setAttribute('p', '生成一个用户资料卡');

// 移除 → 自动取消生成
el.remove();

📖 API 参考

configure(config)

设置全局 LLM 请求函数。必须在任何 <ai-component> 连接之前调用。

参数类型说明
config.request LLMRequestFn 必填。LLM 请求函数，必须返回 AsyncIterable<string>
config.systemPrompt string? 自定义系统提示词，覆盖内置默认值

defineAIComponent()

注册 <ai-component> 自定义元素到 customElements。防止重复注册。使用 IIFE 产物时会自动调用。 

import { defineAIComponent } from 'ai-components';
defineAIComponent();  // 只需调用一次

streamLLM(prompt, signal?)

底层流式生成函数。返回 AsyncGenerator<string>，逐 chunk yield LLM 输出文本。可脱离组件独立使用。

参数类型说明
promptstring用户提示词
signalAbortSignal?中断信号，每次 yield 前检查
 
import { streamLLM } from 'ai-components';

const ctrl = new AbortController();
for await (const chunk of streamLLM('写一首诗', ctrl.signal)) {
  console.log(chunk);
}
// ctrl.abort() 可随时中断

IncrementalDOMBuilder

核心增量 DOM 构建引擎。接收流式 HTML 文本块，通过 createElement / appendChild 逐步构建 DOM 节点。也可独立使用。

方法签名说明
constructor new (root: HTMLElement) 创建构建器，指定根容器元素
write (chunk: string) => void 输入一块 HTML 文本，完整标签立即构建 DOM，不完整标签缓冲等待
finish () => void 结束构建：刷新缓冲区，自动关闭所有未闭合标签。之后 write() 为空操作 
import { IncrementalDOMBuilder } from 'ai-components';

const root = document.getElementById('container');
const builder = new IncrementalDOMBuilder(root);

// 模拟流式输入
builder.write('<div class="card">');
builder.write('<h2>Hello</h2>');
builder.write('<p>World</p>');
builder.write('</div>');
builder.finish();

类型定义

/** LLM 请求函数签名 */
type LLMRequestFn = (
  prompt: string,
  options?: LLMRequestOptions,
) => AsyncIterable<string>;

/** 请求选项 */
interface LLMRequestOptions {
  signal?: AbortSignal;       // 中断信号
  systemPrompt?: string;      // 系统提示词
}

/** 全局配置 */
interface AIComponentsConfig {
  request: LLMRequestFn;      // 必填 — LLM 请求函数
  systemPrompt?: string;      // 自定义系统提示词
}

💬 系统提示词

组件内置了一个经过优化的系统提示词，指导 LLM 输出适合流式渲染的 HTML 片段。你也可以通过 configure({ systemPrompt }) 覆盖。

📋 核心规则：
  • 只输出 HTML 片段，不要 <!DOCTYPE>、<html>、<head>、<body>
  • 使用 Tailwind CSS 类名（假设父页面已加载 CDN）
  • 响应式布局、视觉精美、语义化 HTML
  • 禁止入场动画（流式渲染会导致动画重复触发闪烁）
  • 允许 hover 过渡和持续环境动画
  • 允许 ≤20 行的内联 <script>（必须用 var）
  • 不输出 markdown 围栏

🚨 错误处理

未配置 request 函数
组件显示红色错误边框和提示文字。控制台抛出 [ai-components] LLM request function not configured。
API 请求失败
request 函数抛出的错误会被捕获，组件切换到错误状态并显示 error.message。
LLM 未返回内容
如果流结束但没有收到任何 HTML 标签，显示 "No content generated" 空状态。
用户中断
通过 AbortController 中断生成时，组件保留已渲染的内容不清除。
