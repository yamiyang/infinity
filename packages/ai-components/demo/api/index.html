API 参考

所有导出函数、类和类型的完整文档。

configure(config)

设置全局 LLM 请求函数。必须在任何 AI 组件连接之前调用。所有 AI 组件共享此配置。

参数类型说明
config.request LLMRequestFn 必填。LLM 请求函数，必须返回 AsyncIterable<string>
config.systemPrompt string? 自定义系统提示词，覆盖内置默认值

defineAIComponent()

注册 <ai-component> 自定义元素到 customElements。防止重复注册。使用 IIFE 产物时会自动调用。 

import { defineAIComponent } from 'ai-components';
defineAIComponent();  // 只需调用一次

streamLLM(prompt, signal?, depth?)

底层流式生成函数。返回 AsyncGenerator<string>，逐 chunk yield LLM 输出文本。可脱离组件独立使用。

参数类型说明
promptstring用户提示词
signalAbortSignal?中断信号，每次 yield 前检查
depthnumber?嵌套层级，影响系统提示词策略
 
import { streamLLM } from 'ai-components';

const ctrl = new AbortController();
for await (const chunk of streamLLM('写一首诗', ctrl.signal)) {
  console.log(chunk);
}
// ctrl.abort() 可随时中断

IncrementalDOMBuilder

核心增量 DOM 构建引擎。接收流式 HTML 文本块，通过 createElement / appendChild 逐步构建 DOM 节点。也可独立使用。 构建出的 Custom Element 节点会自动触发其 connectedCallback——这就是嵌套能力的底层实现。

方法签名说明
constructor new (root: HTMLElement) 创建构建器，指定根容器元素
write (chunk: string) => void 输入一块 HTML 文本，完整标签立即构建 DOM，不完整标签缓冲等待
finish () => void 结束构建：刷新缓冲区，自动关闭所有未闭合标签 
import { IncrementalDOMBuilder } from 'ai-components';

const root = document.getElementById('container');
const builder = new IncrementalDOMBuilder(root);

// 模拟流式输入
builder.write('<div class="card">');
builder.write('<h2>Hello</h2>');
builder.write('<ai-component p="嵌套的子组件"></ai-component>');
builder.write('</div>');
builder.finish();

类型定义

/** LLM 请求函数签名 */
type LLMRequestFn = (
  prompt: string,
  options?: LLMRequestOptions,
) => AsyncIterable<string>;

/** 请求选项 */
interface LLMRequestOptions {
  signal?: AbortSignal;       // 中断信号
  systemPrompt?: string;      // 系统提示词
}

/** 全局配置（所有 AI 组件共享） */
interface AIComponentsConfig {
  request: LLMRequestFn;      // 必填 — LLM 请求函数
  systemPrompt?: string;      // 自定义系统提示词
}

💬 系统提示词

每个 AI 组件共享一个系统提示词，指导 LLM 输出适合流式渲染的 HTML 片段。你可以通过 configure({ systemPrompt }) 覆盖。 系统提示词会根据组件的嵌套深度自动调整策略。

📋 核心规则：
  • 只输出 HTML 片段，不要 <!DOCTYPE>、<html>、<head>、<body>
  • 使用 Tailwind CSS 类名（假设父页面已加载 CDN）
  • 响应式布局、视觉精美、语义化 HTML
  • 禁止入场动画（流式渲染会导致动画重复触发闪烁）
  • 允许 hover 过渡和持续环境动画
  • 允许 ≤20 行的内联 <script>（必须用 var）
  • 不输出 markdown 围栏
  • 可以在输出中使用 <ai-component> 标签来将子任务委托给嵌套的 AI 组件

🚨 错误处理

未配置 request 函数
组件显示红色错误边框和提示文字。控制台抛出 [ai-components] LLM request function not configured。
API 请求失败
request 函数抛出的错误会被捕获，组件切换到错误状态并显示 error.message。
LLM 未返回内容
如果流结束但没有收到任何 HTML 标签，显示 "No content generated" 空状态。
用户中断
通过 AbortController 中断生成时，组件保留已渲染的内容不清除。
嵌套深度超限
超过 MAX_DEPTH (3) 的组件不会发起 LLM 请求，显示 "Maximum nesting depth exceeded" 警告。
