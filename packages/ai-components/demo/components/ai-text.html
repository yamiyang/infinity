<ai-text>

AI 纯文本组件 — 流式生成纯文本，无 HTML 解析

<ai-text> 是一个纯文本生成组件。与 <ai-component> 不同的是，它只输出纯文本内容（使用 textContent），不会解析或渲染任何 HTML 标签。 适合用于标题、段落、广告文案、诗歌、翻译等场景。

纯文本 流式输出 XSS 安全 支持 ai-data

💡 与 ai-component 的区别： <ai-text> 使用 textContent 追加内容，天然防止 XSS， 也不会因为 LLM 输出意外的 HTML 标签而破坏页面布局。当你只需要文本内容时，优先选择它。

快速开始

<!-- 最简用法 -->
<ai-text p="写一首关于春天的俳句"></ai-text>

<!-- 自定义标签和样式 -->
<ai-text p="用一句话描述量子计算" tag="span" class="text-lg font-bold text-indigo-600"></ai-text>

<!-- 搭配 ai-data 使用 -->
<ai-text p="给这个产品写一段广告语">
  <ai-data p="mock一个智能手表的产品信息" />
</ai-text>

属性

属性 类型 默认值 说明
p string — 必填。自然语言提示词
tag string "p" 包裹元素的标签名。可用 "span"、"h1"、"div" 等
class string — 转发到内部包裹元素的 CSS 类名
style string — 转发到内部包裹元素的行内样式

使用示例

基础文本

最简单的用法 — 一个 prompt 生成一段文字。

<ai-text p="写一首关于秋天的五言绝句"></ai-text>

行内文本

使用 tag="span" 生成行内文本，嵌入到其他内容中。

<p>今日天气：<ai-text p="用3个字描述今天北京的天气" tag="span"></ai-text></p>

<h2>
  <ai-text p="给一个科技博客起一个吸引人的名字" tag="span" class="text-indigo-600"></ai-text>
</h2>

搭配 ai-data

先获取产品数据，再基于数据生成文案。

<ai-text p="用诗意的语言描述这个产品的核心卖点，不超过50字">
  <ai-data p="mock一个高端降噪耳机的产品信息" />
</ai-text>

动态标题

用 tag="h1" 生成页面标题。

<ai-text p="给一个在线教育平台起一个有创意的标题" tag="h1"
  class="text-4xl font-extrabold bg-gradient-to-r from-indigo-600 to-purple-600 bg-clip-text text-transparent">
</ai-text>

组合布局

多个 ai-text 组合使用，构建完整的文本内容。

<div class="space-y-4">
  <ai-text p="给科技文章起标题" tag="h2" class="text-2xl font-bold"></ai-text>
  <ai-text p="写一段100字左右的科技文章开头" class="text-gray-600 leading-relaxed"></ai-text>
  <ai-text p="用一句话做文章总结" tag="blockquote" class="border-l-4 border-indigo-400 pl-4 italic text-gray-500"></ai-text>
</div>

工作原理

1
等待数据 — 如果内部有 <ai-data> 子元素，先等待其解析完成。
2
专用系统提示词 — 使用纯文本专用的系统提示词，明确禁止 LLM 输出 HTML / Markdown 标记。
3
textContent 追加 — 使用 textContent 而非 innerHTML 追加内容。即使 LLM 意外输出 HTML，也只会显示为纯文本。
4
可定制标签 — 通过 tag 属性选择包裹元素（<p>、<span>、<h1> 等），class 和 style 自动转发。

对比 ai-component

特性 <ai-text> <ai-component>
输出类型 纯文本 HTML 片段
DOM 构建 textContent IncrementalDOMBuilder
XSS 安全 ✅ 天然安全 ⚠️ 需信任 LLM 输出
嵌套支持 不支持嵌套自身 3 层深度
适用场景 标题、文案、翻译 完整页面/组件
