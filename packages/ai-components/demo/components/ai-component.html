<ai-component>

可用

组件库的核心通用组件——接受任意自然语言 prompt，调用 LLM 流式生成并实时渲染 HTML 片段。 支持递归嵌套和深度感知（最多 3 层），每层自动调整生成策略。

📖 文档
🎮 Playground

⚙️ 属性（Attributes）

属性类型默认值说明
p string — 必填。自然语言提示词。设置后组件自动调用 LLM 并流式渲染结果。修改后重新生成。
data-depth number 自动计算 只读。组件自动设置的嵌套层级（1-3），可用于 CSS 选择器或调试。

🔄 生命周期

connected 组件挂载时计算嵌套层级（depth），读取 p 属性并自动开始生成。先显示加载态（spinner + prompt 预览），流开始后切换到内容模式。
attribute p 属性变化时，取消当前生成并重新执行。
disconnected 从 DOM 移除时自动取消正在进行的生成（通过 AbortController）。

📊 嵌套 & 深度感知

<ai-component> 会自动检测自己在 DOM 树中的嵌套层级，并据此调整系统提示词中的生成策略。

层级角色行为
depth 1 骨架编排器 生成布局骨架（grid / flex），大量使用 <ai-component> 委派子任务
depth 2 段落构建器 渲染具体内容段落，可适度再分配 1-2 个子组件
depth 3 叶子节点 纯展示内容，禁止再嵌套 <ai-component>
> 3 阻断 不发起 LLM 请求，显示 "Maximum nesting depth exceeded" 警告

💡 使用示例

基本用法

<!-- 最简单的用法 -->
<ai-component p="一个天气预报卡片"></ai-component>

动态创建

<script>
  const el = document.createElement('ai-component');
  el.setAttribute('p', '生成一个用户资料卡');
  document.body.appendChild(el);

  // 更新 prompt → 自动重新生成
  el.setAttribute('p', '改成暗色主题的用户资料卡');

  // 移除 → 自动取消生成
  el.remove();
</script>

嵌套使用（自动深度感知）

<!-- depth 1：编排器模式，自动生成布局 + 委派子组件 -->
<ai-component p="生成一个仪表盘布局，用 ai-component 分别展示统计卡片、图表和列表"></ai-component>

<!-- LLM 可能输出类似这样的结构 -->
<div class="grid grid-cols-2 gap-4">
  <!-- 这些子组件自动成为 depth 2（段落构建器）-->
  <ai-component p="用户统计卡片"></ai-component>
  <ai-component p="收入趋势图表"></ai-component>
  <ai-component p="最近订单列表"></ai-component>
  <ai-component p="系统通知"></ai-component>
</div>

CSS 按层级定制样式

/* 利用自动设置的 data-depth 属性 */
ai-component[data-depth="1"] { border: 2px solid #6366f1; }
ai-component[data-depth="2"] { border: 1px solid #a5b4fc; }
ai-component[data-depth="3"] { border: 1px dashed #c7d2fe; }
