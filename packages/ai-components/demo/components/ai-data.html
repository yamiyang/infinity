<ai-data>

AI 数据提供组件 — 异步获取数据，驱动其他组件

<ai-data> 是一个不渲染任何内容的基础设施组件。 它通过 LLM 异步获取结构化数据，并将数据注入到父 AI 组件的 prompt 中。 所有 AI 组件（<ai-component>、<ai-image>、<ai-map> 等） 都会在启动自己的生命周期前，先等待内部的 <ai-data> 子元素全部就绪。

🔗

数据驱动

先获取数据，再渲染 UI。分离数据层与展示层。

🪆

可嵌套

ai-data 可以嵌套 ai-data，内层先完成，外层拿到内层数据后再执行。

👻

零渲染

display: none，不产生任何可见输出。纯数据层。

核心概念

数据流：ai-data → 父组件

1. 浏览器解析 HTML → 触发 <ai-data> 的 connectedCallback
2. <ai-data> 检查自身是否有嵌套的 <ai-data> 子元素
├─ 有 → 先等待子 <ai-data> 全部就绪
└─ 无 → 直接开始
3. 调用 LLM，用数据专用系统提示词，获取纯数据（JSON/text/CSV）
4. 清理响应 → 存储到 this.data → 触发 'ai-data-ready' 事件
5. 父组件收到事件 → 将数据注入自己的 prompt → 开始自身生命周期

阻塞机制

当一个 AI 组件（如 <ai-component>）检测到内部有 <ai-data> 子元素时， 它会暂停自己的 LLM 调用，等待所有直接子 <ai-data> 元素发出 ai-data-ready 事件后， 才将收集到的数据拼接到自己的 prompt 中并开始生成。

如果没有 <ai-data> 子元素，组件行为与之前完全一致 — 零开销。

属性 (Attributes)

属性 类型 默认值 说明
p string — 必填。自然语言描述需要的数据内容。
示例："mock一个购物车列表数据，包含商品名、价格、数量"
format string "json" 输出格式提示："json"、"text"、"csv" 等。
LLM 默认输出 JSON，通过此属性可切换格式。

JavaScript 属性

属性 类型 说明
.data string | null 数据就绪后可通过此属性同步读取解析后的数据字符串。

事件

事件名 detail 类型 说明
ai-data-ready string 数据成功获取后触发。event.detail 为解析后的数据字符串。
ai-data-error string 数据获取失败时触发。event.detail 为错误消息。

使用示例

1. 基本用法 — 为 AI 组件提供数据

数据组件先获取购物车数据，然后 <ai-component> 拿到数据后再生成 UI。

<ai-component p="渲染一个购物车列表页面">
  <ai-data p="mock一个购物车数据，5件商品，包含名称、价格、数量、图片URL" />
</ai-component>

2. 多个数据源

一个组件可以包含多个 <ai-data>，所有数据全部就绪后才渲染。

<ai-component p="用户个人主页，展示用户信息和最近订单">
  <ai-data p="mock一个用户的个人资料：头像、姓名、邮箱、会员等级" />
  <ai-data p="mock该用户最近5笔订单：订单号、商品、金额、状态" />
</ai-component>

3. 嵌套数据 — ai-data 嵌套 ai-data

内层 ai-data 先获取基础数据，外层再基于它生成衍生数据。

<ai-component p="展示产品详情页">
  <ai-data p="根据以下产品基础信息，生成完整的产品详情数据，包括 SEO 描述、推荐理由、用户评价">
    <ai-data p="mock一个电子产品的基础信息：名称、品牌、价格、规格参数" />
  </ai-data>
</ai-component>

4. 配合其他 AI 组件

<ai-data> 可以配合任意 AI 组件使用，不仅限于 <ai-component>。

<!-- 根据数据搜索对应图片 -->
<ai-image p="展示这个城市的标志性景观">
  <ai-data p="随机选一个世界著名城市，返回城市名" format="text" />
</ai-image>

<!-- 根据数据在地图上标注 -->
<ai-map p="在地图上标注这些地点">
  <ai-data p="列出巴黎最著名的5个旅游景点及其地址" />
</ai-map>

<!-- 根据数据播放对应音乐 -->
<ai-music p="播放适合这个场景的音乐">
  <ai-data p="描述一个下雨天在咖啡馆阅读的场景氛围" format="text" />
</ai-music>

5. 指定数据格式

通过 format 属性指定 LLM 返回的数据格式。

<!-- CSV 格式 -->
<ai-component p="用表格展示以下 CSV 数据">
  <ai-data p="mock一份销售报表数据，10行，包含日期、产品、销量、营收" format="csv" />
</ai-component>

<!-- 纯文本格式 -->
<ai-component p="用优美的排版展示这首诗">
  <ai-data p="写一首关于春天的五言绝句" format="text" />
</ai-component>

6. JavaScript 编程式访问

可以通过 JavaScript 监听事件或读取 .data 属性来获取数据。

<ai-data id="myData" p="生成5个随机用户数据"></ai-data>

<script>
  var dataEl = document.getElementById('myData');

  // 方式1: 监听事件
  dataEl.addEventListener('ai-data-ready', function(e) {
    console.log('数据已就绪:', e.detail);
    var users = JSON.parse(e.detail);
    // 用数据做任何事情...
  });

  // 方式2: 轮询检查
  // dataEl.data 在就绪后为非 null 字符串
</script>

工作原理

1

连接到 DOM

connectedCallback 触发，设置 display: none，开始数据获取流程。

2

等待子数据（如有嵌套）

检查内部是否有嵌套的 <ai-data> 子元素。如果有，先等待它们全部完成并收集数据。

3

调用 LLM 获取数据

使用数据专用系统提示词（要求只输出纯数据，无 Markdown），将子数据作为上下文注入 prompt，流式获取 LLM 响应并累积。

4

清理 & 存储

去除可能的代码围栏标记，清理首尾空白。将结果存储到 .data 属性。

5

触发就绪事件

派发 ai-data-ready CustomEvent（不冒泡），父组件收到后将数据注入自己的 prompt 并启动生成。

架构设计

通用基类能力

<ai-data> 的等待机制不是写在 ai-data 组件内部的，而是通过核心工具函数 waitForChildData() 实现的。这意味着所有 AI 组件都自动获得了这个能力 — 只需在自己的生命周期开始前调用一下这个函数即可。 

// 核心工具函数（src/core/data-manager.ts）
import { waitForChildData, buildPromptWithData } from './core/data-manager';

// 在任意 AI 组件的 _generate / _render 方法中：
const childData = await waitForChildData(this, signal);
const finalPrompt = buildPromptWithData(prompt, childData);

如果没有 <ai-data> 子元素，waitForChildData() 立即返回 null，零开销。
